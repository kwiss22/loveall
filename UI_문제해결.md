# UI가 안 나오는 문제 해결

## 🔍 문제 진단

웹 애플리케이션이 실행되지만 UI가 안 보이는 경우, 대부분 CSS나 JavaScript 파일이 제대로 로드되지 않았을 때 발생합니다.

---

## ✅ 해결 방법

### 방법 1: 브라우저 하드 리프레시 (가장 간단)

1. **브라우저에서 Ctrl + Shift + R** (하드 리프레시)
   - 또는 **Ctrl + F5**
   - 또는 **F12** 키 → 새로고침 버튼 오른쪽 클릭 → "캐시 비우기 및 강력 새로고침"

2. **브라우저 주소 확인:**
   ```
   http://127.0.0.1:5000
   ```
   - `/main` 같은 경로가 붙어있지 않은지 확인

### 방법 2: 브라우저 개발자 도구로 확인

1. **F12 키** 눌러서 개발자 도구 열기

2. **Console 탭 확인:**
   - 빨간색 오류 메시지가 있는지 확인
   - CSS나 JS 파일 로드 오류가 있는지 확인

3. **Network 탭 확인:**
   - F5로 새로고침
   - `style.css` 파일이 로드되는지 확인
   - 상태가 **200 OK**인지 확인
   - 상태가 **404 Not Found**이면 파일 경로 문제

### 방법 3: CSS 파일 직접 확인

브라우저 주소창에 직접 입력:
```
http://127.0.0.1:5000/static/style.css
```

**결과 확인:**
- ✅ CSS 코드가 보이면 정상 (파일 경로는 올바름)
- ❌ "Not Found" 오류가 나면 Flask의 static 폴더 설정 문제

### 방법 4: Flask 앱 재시작

1. **터미널에서 Ctrl + C**로 Flask 앱 종료

2. **다시 실행:**
   ```powershell
   python web_vocab_app.py
   ```

3. **브라우저에서 Ctrl + Shift + R** (하드 리프레시)

### 방법 5: 브라우저 캐시 완전히 지우기

1. **Ctrl + Shift + Delete** 눌러서 캐시 삭제
2. **브라우저 닫기**
3. **새 브라우저 창 열기**
4. **주소 입력:** `http://127.0.0.1:5000`

## 🔧 추가 확인 사항

### 1. 올바른 Flask 앱이 실행 중인가?

터미널에서 다음 메시지가 보여야 합니다:
```
* Running on http://127.0.0.1:5000
* Debug mode: on
```

**중요:** `web_vocab_app.py`가 실행 중이어야 합니다!
- ❌ `test_flask_simple.py` (X) - 이것은 테스트용
- ✅ `web_vocab_app.py` (O) - 이것이 실제 앱

### 2. 올바른 주소로 접속했나?

- ✅ `http://127.0.0.1:5000` (O)
- ❌ `http://127.0.0.1:5000/main` (X)

### 3. 파일 구조 확인

다음 파일들이 있어야 합니다:
```
game_english/
├── web_vocab_app.py
├── templates/
│   └── index.html
└── static/
    ├── style.css  ← 이 파일이 있어야 합니다!
    └── script.js  ← 이 파일도 있어야 합니다!
```

## 💡 단계별 해결 순서

### 1단계: Flask 앱 확인
```powershell
python web_vocab_app.py
```
- "Running on http://127.0.0.1:5000" 메시지 확인

### 2단계: 브라우저 하드 리프레시
- **Ctrl + Shift + R**

### 3단계: 개발자 도구 확인
- **F12** 키
- **Console 탭** 확인 (빨간색 오류 확인)
- **Network 탭** 확인 (`style.css` 로드 확인)

### 4단계: CSS 파일 직접 접근
- 브라우저 주소창: `http://127.0.0.1:5000/static/style.css`
- CSS 코드가 보이는지 확인

### 5단계: 문제가 계속되면
- Flask 앱 재시작
- 브라우저 캐시 삭제
- 다른 브라우저로 시도

---

## ✅ 정상 작동 확인

CSS가 제대로 로드되면:
- **보라색 그라데이션 배경**이 보여야 합니다
- **흰색 카드 형태의 컨테이너**가 보여야 합니다
- **보라색 헤더**가 보여야 합니다
- **탭 버튼** (단어 목록, 단어 추가, 퀴즈, 통계)이 보여야 합니다
- **예쁘게 디자인된 화면**이 보여야 합니다

---

## 🔍 일반적인 원인

1. **브라우저 캐시 문제** → 하드 리프레시
2. **CSS 파일 경로 문제** → Flask의 static 폴더 확인
3. **잘못된 Flask 앱 실행** → `web_vocab_app.py` 확인
4. **파일이 없음** → `static/style.css` 확인
