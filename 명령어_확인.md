# 올바른 명령어 사용법

## ❌ 잘못된 명령어

```
web_vocab_app.py
```

PowerShell에서는 이것을 인식할 수 없습니다!

## ✅ 올바른 명령어

```
python web_vocab_app.py
```

**중요:** `python`을 앞에 붙여야 합니다!

---

## 📝 단계별 실행 방법

### 1단계: 현재 실행 중인 앱 확인

터미널 상단을 보면 Flask 앱이 이미 실행 중인 것 같습니다:
```
* Running on http://127.0.0.1:5000
* Debug mode: on
```

### 2단계: 이전 앱이 실행 중이라면

**옵션 1: 현재 실행 중인 앱 사용**
- 터미널에서 **Ctrl + C**를 눌러 중지
- 올바른 명령어로 다시 실행

**옵션 2: 새로운 터미널 창 열기**
- 새 PowerShell 창 열기
- 명령어 실행

### 3단계: 올바른 명령어 실행

PowerShell에서:
```powershell
python web_vocab_app.py
```

**주의:**
- `python`을 반드시 앞에 붙여야 합니다!
- `web_vocab_app.py`만 입력하면 안 됩니다!

### 4단계: 성공 메시지 확인

다음 메시지가 보이면 성공:
```
============================================================
영어 단어장 웹 애플리케이션 시작!
============================================================
단어 개수: 6개
통계 기록: 2개
접속 주소: http://127.0.0.1:5000
============================================================

* Running on http://127.0.0.1:5000
```

## 🔍 명령어 비교

### Python 파일 실행

| 명령어 | 결과 |
|--------|------|
| `web_vocab_app.py` | ❌ 오류 발생 |
| `python web_vocab_app.py` | ✅ 정상 실행 |

### 다른 예시

| 명령어 | 결과 |
|--------|------|
| `vocab_book.py` | ❌ 오류 |
| `python vocab_book.py` | ✅ 정상 |
| `test.py` | ❌ 오류 |
| `python test.py` | ✅ 정상 |

---

## 💡 팁

1. **항상 `python` 명령어 사용:**
   - Python 파일을 실행할 때는 항상 `python`을 앞에 붙입니다
   - `python 파일명.py`

2. **PowerShell vs 명령 프롬프트:**
   - 둘 다 `python` 명령어를 사용합니다
   - 명령어는 동일합니다

3. **파일 이름만으로는 실행 안 됨:**
   - Windows에서는 Python 파일을 직접 실행할 수 없습니다
   - 반드시 `python` 명령어를 사용해야 합니다
