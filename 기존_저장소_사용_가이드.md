# 기존 GitHub 저장소 사용 가이드

## ✅ 기존 저장소 사용 가능!

기존에 만든 GitHub 저장소를 사용할 수 있습니다. 새로 만들 필요 없이 기존 저장소에 연결하면 됩니다.

---

## 🚀 기존 저장소에 연결하는 방법

### 방법 1: 빈 저장소인 경우 (추천)

기존 저장소가 비어있거나 README만 있는 경우:

```bash
# 1. Git 사용자 정보 설정 (아직 안 했다면)
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 2. 초기 커밋 생성
git commit -m "Initial commit: 영어 단어장 웹 애플리케이션"

# 3. 기존 저장소 연결 (YOUR_USERNAME과 REPO_NAME을 실제 값으로 변경)
git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git

# 4. 브랜치 이름 확인 및 변경 (필요시)
git branch -M main

# 5. GitHub에 업로드
git push -u origin main
```

### 방법 2: 저장소에 이미 파일이 있는 경우

기존 저장소에 파일이 있다면:

```bash
# 1. 기존 저장소 연결
git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git

# 2. 기존 파일 가져오기
git pull origin main --allow-unrelated-histories

# 3. 충돌 해결 (있는 경우)
# 4. 커밋
git commit -m "Initial commit: 영어 단어장 웹 애플리케이션"

# 5. 업로드
git push -u origin main
```

## 📋 단계별 가이드

### 1단계: 저장소 URL 확인

GitHub에서 기존 저장소를 열고:
- 저장소 페이지에서 초록색 "Code" 버튼 클릭
- HTTPS URL 복사 (예: `https://github.com/kwiss22/game_english.git`)

### 2단계: 로컬 저장소와 연결

```bash
# 원격 저장소 추가
git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git

# 연결 확인
git remote -v
```

### 3단계: Git 사용자 정보 설정 (필수)

```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

### 4단계: 커밋 생성

```bash
git commit -m "Initial commit: 영어 단어장 웹 애플리케이션"
```

### 5단계: GitHub에 업로드

```bash
# 브랜치 이름 확인
git branch

# main으로 변경 (필요시)
git branch -M main

# 업로드
git push -u origin main
```

## ⚠️ 주의사항

### 저장소가 비어있지 않은 경우

기존 저장소에 파일이 있다면:
- `--allow-unrelated-histories` 옵션 사용
- 충돌 발생 시 수동으로 해결 필요
- 또는 기존 파일을 백업 후 덮어쓰기

### 이미 원격 저장소가 연결된 경우

```bash
# 기존 원격 저장소 확인
git remote -v

# 기존 원격 저장소 제거
git remote remove origin

# 새 원격 저장소 추가
git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git
```

## 💡 추천 방법

### 가장 안전한 방법:

1. **기존 저장소가 비어있는 경우**
   - 그냥 연결하고 push

2. **기존 저장소에 파일이 있는 경우**
   - 기존 파일을 백업
   - 또는 새 브랜치 생성 후 작업

---

## 🔍 저장소 확인 방법

GitHub에서:
1. 저장소 페이지 접속
2. 파일 목록 확인
3. README.md가 있는지 확인
4. 다른 파일이 있는지 확인
