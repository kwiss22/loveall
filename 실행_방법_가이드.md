# 애플리케이션 실행 방법 가이드

## 🌐 방법 1: 배포된 웹사이트 접속 (가장 간단!)

Railway에 배포된 웹사이트에 접속하는 방법입니다.

### 1단계: Railway 대시보드 접속
1. 브라우저에서 https://railway.app 접속
2. 로그인
3. `loveall` 프로젝트 클릭

### 2단계: 서비스 URL 확인
1. **Settings → Domains** 탭 클릭
   - 또는 **Deployments** 탭에서 최신 배포 클릭
   - "View Logs" 또는 "View Service" 클릭

2. **도메인 확인**
   - 자동 생성된 도메인 확인 (예: `loveall-production.up.railway.app`)
   - 또는 "Generate Domain" 버튼 클릭하여 새 도메인 생성

### 3단계: 웹사이트 접속
1. 생성된 URL 클릭하거나 복사
2. 브라우저에서 접속
3. 애플리케이션 사용!

## 💻 방법 2: 로컬에서 실행 (개발용)

로컬 컴퓨터에서 직접 실행하는 방법입니다.

### 1단계: 터미널에서 프로젝트 디렉토리로 이동
```bash
cd C:\Users\Admin\projects\game_english
```

### 2단계: 가상 환경 활성화 (선택사항)
```bash
# 가상 환경이 있다면
.\venv\Scripts\activate

# 또는
python -m venv venv
.\venv\Scripts\activate
```

### 3단계: 의존성 설치 (필요시)
```bash
pip install -r requirements.txt
```

### 4단계: 애플리케이션 실행
```bash
python web_vocab_app.py
```

### 5단계: 브라우저에서 접속
- 터미널에 표시된 주소로 접속
- 기본 주소: http://127.0.0.1:5000
- 또는 http://localhost:5000

## 🚀 빠른 실행 (Windows)

### 배치 파일 사용 (가장 간단!)

프로젝트에 `run_web_app.bat` 파일이 있다면:

1. **파일 탐색기**에서 `run_web_app.bat` 더블클릭
2. 자동으로 서버가 시작됨
3. 브라우저에서 http://127.0.0.1:5000 접속

## 🔍 문제 해결

### 로컬 실행 시 포트가 사용 중이면?
```
OSError: [Errno 48] Address already in use
```

**해결책:**
1. 다른 포트 사용:
   ```bash
   python web_vocab_app.py --port 5001
   ```
2. 또는 기존 프로세스 종료:
   ```bash
   # Windows PowerShell
   netstat -ano | findstr :5000
   taskkill /PID <프로세스ID> /F
   ```

### 의존성 설치 오류?
```bash
# pip 업그레이드
python -m pip install --upgrade pip

# requirements.txt 재설치
pip install -r requirements.txt
```

### 가상 환경 문제?
```bash
# 새 가상 환경 생성
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
```

## ✅ 추천 방법

### 일반 사용자
→ **배포된 웹사이트 접속** (방법 1)
- Railway URL로 바로 접속
- 별도 설치 불필요
- 어디서나 접속 가능

### 개발자/테스트
→ **로컬 실행** (방법 2)
- 빠른 개발/테스트
- 코드 수정 즉시 반영
- 오프라인 사용 가능
